## 3. 修过的业务 bug

### `update_contract` 金额处理不一致

**问题**：
- `create_contract`: `c.amount = req.amount * 100`（元→分，✓ 正确）
- `update_contract`: `c.amount = req.amount`（**未乘 100**，✗ 错误）

**影响**：用户 update 合同金额时，DB 存储少了 100 倍。

**修复**（`backend/app/modules/contract/service.py:202-203`）：
```python
elif k == "amount":
    # 与 create_contract 保持一致：API 入参是元，DB 存分（×100）
    c.amount = v * 100
```

**测试覆盖**：`test_update_success_draft` 单跑通过，全量跑受 Event loop 干扰。

## 4. 仍未解决的问题

### 4.1 测试间 Event loop 干扰（3 个 fail）

**症状**：3 个测试（`test_update_success_draft` / `test_submit_draft_success` / `test_stats_pending_count`）**单跑全过**，**全量跑**在第 19 个 test 开始崩 `RuntimeError: Event loop is closed`。

**已尝试的修法（都失败）**：
- ✗ `NullPool` 替换 pool
- ✗ 重写 `database.AsyncSessionLocal` 指向测试 session
- ✗ 改 `_test_db_setup` 为 function-scope
- ✗ audit middleware session 注入重写
- ✗ ASGITransport 替换 httpx(app=app)

**真因（推测）**：pytest-asyncio 0.23 + asyncpg 在 async 上下文切换时，asyncpg 的 connection close callback 仍调用 `loop.call_soon`，但 loop 已 close。

**未来方案**：
- 升级 pytest-asyncio 到 0.23+ 配 anyio 模式
- 或改用 sync SQLAlchemy + pytest-asyncio 同步模式
- 或每次 fixture dispose 时加 `await asyncio.sleep(0)` 让 loop drain

### 4.2 `test_approve_full_flow` IntegrityError

**症状**：审批流测试在事务回滚期间数据未清。

**修法**（下次）：fixture 加 `await db.rollback()` 兜底，或审批流用 savepoint。
